# Contributing

Thank you for considering a contribution! Please follow these guidelines to keep the project consistent and stable.

## Workflow
1. Fork the repository and create a feature branch.
2. Keep changes focused and documented.
3. Run linters/tests where applicable.
4. Submit a pull request with a clear description.

## Coding Standards
- PHP: Modern PHP 8.1 syntax, type declarations, and PSR-12 style.
- Python: PEP 8 with type hints.
- JavaScript: ES modules or organized scripts; prefer const/let.
- Security first: validate input, escape output, and avoid exposing secrets.

## Commit Messages
- Use concise titles (e.g., `Add gallery API pagination`).
- Reference issues when relevant.

## Issue Reporting
- Use the provided issue templates (`bug_report.md`, `feature_request.md`).
- Include reproduction steps, logs, and environment details.

## Code of Conduct
Be respectful and constructive. We build for creators of all backgrounds.
